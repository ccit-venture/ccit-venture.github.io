# Schedule-Based Position Simulator

## Overview

**Produk:** [168Railway](https://168railway.com)  
**URL:** https://168railway.com

168Railway adalah platform tracking kereta api Indonesia yang menyediakan informasi **posisi kereta secara real-time** berdasarkan **jadwal GAPEKA** (Grafik Perjalanan Kereta Api). Platform ini menggunakan **algoritma prediksi** yang menghitung posisi kereta berdasarkan waktu keberangkatan, waktu tiba, dan jarak antar stasiun - **tanpa memerlukan GPS hardware** di kereta maupun di pengguna.

---

## The Mystery

Bagaimana cara membangun platform tracking kereta yang terlihat "real-time" tanpa harus memasang GPS sensor mahal di setiap kereta dan tanpa mengandalkan data GPS dari pengguna?

**Problem:**
- Penumpang kereta api ingin tahu posisi kereta saat ini
- Jadwal GAPEKA tersedia, tapi sulit divisualisasikan secara real-time
- Tidak ada budget untuk IoT/GPS hardware di setiap kereta
- Solusi GPS crowdsourcing tidak feasible (privasi, baterai, adoption rate rendah)

**Solution:**
- **Schedule-Based Simulation**: Prediksi posisi berdasarkan jadwal resmi GAPEKA
- **Interpolation Algorithm**: Hitung posisi di antara stasiun berdasarkan waktu
- **Visualisasi Live**: Tampilkan posisi yang "bergerak" di peta secara smooth
- **Smart Prediction**: Asumsikan kereta jalan on-time (karena memang jarang delay)

## Behind the Scenes

### Cara Kerja 168Railway

> **Jadwal GAPEKA** → **Interpolasi Waktu-Jarak** → **Simulasi Pergerakan** → **Visualisasi Real-Time**

#### Layer 1: Data Sources

**A. GAPEKA Official Data (Sumber Utama)**
- Jadwal resmi dari KAI (Kereta Api Indonesia)
- Waktu keberangkatan dari setiap stasiun
- Waktu tiba di stasiun berikutnya
- Daftar stasiun dan urutannya (rute)
- Jarak antar stasiun

**B. Static Data**
- Profil stasiun (fasilitas, alamat, koordinat GPS)
- Informasi depo lokomotif
- Data teknis kereta (CC 201, CC 206, etc.)
- Foto lokomotif

**C. Manual Updates (Opsional)**
- Input delay manual (kalau ada info delay)
- Status perubahan jadwal
- Cancelled trips

#### Layer 2: The Algorithm

**Simulasi Pergerakan Kereta:**

> **Input:**
> - Stasiun A: Berangkat jam 08:00
> - Stasiun B: Tiba jam 10:00
> - Jarak A-B: 120 km
> - Waktu sekarang: 09:00
> 
> **Perhitungan:**
> - Total waktu perjalanan = 2 jam
> - Waktu yang sudah lewat = 1 jam (50%)
> - Posisi saat ini = 50% dari jarak A-B
> - Jarak dari Stasiun A = 60 km

**Interpolasi Posisi:**
- Dapatkan koordinat GPS Stasiun A dan Stasiun B
- Hitung posisi di garis lurus antara A dan B
- Sesuaikan dengan persentase waktu yang sudah lewat
- Update posisi setiap 30 detik untuk efek "bergerak"

**Smooth Movement:**
- Jangan update posisi langsung (jumpy)
- Gunakan easing function untuk pergerakan halus
- Simulasikan kecepatan realistis (tidak instant teleport)

#### Layer 3: Visualization

**Interactive Map (Leaflet.js):**
- Peta Indonesia dengan rute kereta api
- Marker kereta yang "bergerak" mengikuti jalur
- Popup informasi kereta (no. KA, tujuan, ETA ke stasiun berikutnya)
- Jalur berwarna sesuai status (on-time, delayed, cancelled)

**Dashboard Components:**
- **Live Departure Board**: Seperti di bandara/stasiun
- **Train Cards**: Info singkat kereta aktif dengan posisi simulasi
- **Station Search**: Cari jadwal per stasiun
- **ETA Calculator**: Estimasi waktu tiba berdasarkan posisi simulasi

#### Layer 4: Features

**Core Features:**
1. **Simulated Position Tracking**
   - Peta dengan marker kereta yang bergerak
   - Info kereta: No. KA, jurusan, posisi % dari rute
   - Klik kereta untuk detail lengkap

2. **GAPEKA 2025**
   - Jadwal lengkap seluruh kereta Indonesia
   - Filter berdasarkan rute, waktu, tipe kereta
   - Search stasiun

3. **Station Information**
   - Jadwal keberangkatan per stasiun
   - Fasilitas stasiun
   - Peta lokasi stasiun

4. **Locomotive Database**
   - Profil lokomotif (CC 201, CC 203, CC 206, etc.)
   - Foto dan spesifikasi teknis
   - Status operasional

**Tech Stack:**
- **Frontend**: Laravel + Blade + Alpine.js + Tailwind CSS
- **Map**: Leaflet.js (OpenStreetMap)
- **Algorithm**: JavaScript untuk interpolasi posisi
- **Database**: MySQL untuk jadwal GAPEKA dan data stasiun
- **Scheduling**: Cron job untuk update posisi otomatis

## The Simple Truth

**"Kamu gak perlu GPS hardware untuk tracking - cukup jadwal dan matematika sederhana!"**

Kenapa 168Railway works:
1. **No Hardware Required**: Gak perlu pasang sensor mahal di kereta
2. **High Accuracy**: Kereta api Indonesia jarang delay (kecuali ada masalah jalur)
3. **Schedule-Driven**: GAPEKA adalah sumber truth yang reliable
4. **Visual Impact**: Pengguna melihat kereta "bergerak" di peta = feels real-time
5. **Cost Effective**: Solusi software, bukan hardware

**Pattern: Schedule-Based Position Simulation**
- Gunakan jadwal sebagai dasar prediksi
- Interpolasi posisi berdasarkan waktu
- Asumsikan on-time (valid untuk sistem yang reliable)
- Visualisasi yang meyakinkan

## Key Insight

### The "Schedule-Based Simulation Pattern"

**Use fixed schedule + interpolation algorithm → Simulate position → Display as real-time tracking**

#### Kenapa Pattern Ini Efektif?

1. **Zero Hardware Cost**
   - Gak perlu beli GPS tracker
   - Gak perlu maintenance hardware
   - Gak perlu deal dengan operator transportasi untuk pasang sensor

2. **High Reliability**
   - Kereta api (dan transportasi scheduled lainnya) punya jadwal yang ketat
   - Delay jarang terjadi (kecuali force majeure)
   - Data jadwal tersedia dan akurat

3. **Scalable**
   - Tambah rute baru = cukup input jadwal
   - Tidak bergantung pada jumlah pengguna
   - Bisa cover semua kereta, bukan cuma yang ada penggunanya

4. **User Experience**
   - Pengguna lihat posisi "real-time" (meskipun simulated)
   - ETA akurat karena based on schedule
   - Visualisasi membantu planning perjalanan

#### When to Use This Pattern?

✅ **Use:**
- Transportasi dengan jadwal fixed (kereta, bus antar kota)
- Sistem yang on-time rate tinggi (>90%)
- Dimana hardware tracking mahal/tidak feasible
- Prototype/MVP sebelum investasi hardware

❌ **Don't Use:**
- Transportasi tanpa jadwal fixed (ojek online, taksi)
- Sistem dengan delay rate tinggi
- Use case yang butuh precision meter-level
- Tracking yang butuh real-time updates saat delay terjadi

## Simple Version You Can Build

### MVP: Campus Shuttle Tracker

**Konsep:** Tracking shuttle bus kampus dengan simulasi berbasis jadwal

**Data yang Dibutuhkan:**

```plaintext
{
  "route": "Fakultas Teknik → Fakultas Ekonomi",
  "stops": [
    {"name": "Fakultas Teknik", "departure": "08:00", "lat": -6.123, "lng": 106.456},
    {"name": "Perpustakaan", "arrival": "08:05", "departure": "08:07", "lat": -6.124, "lng": 106.457},
    {"name": "Fakultas Ekonomi", "arrival": "08:15", "lat": -6.125, "lng": 106.458}
  ]
}
```

**Algorithm Sederhana:**

```plaintext
function calculateBusPosition(schedule, currentTime) {
  // Cari segmen (antar 2 stop) yang sedang ditempuh
  const currentSegment = findCurrentSegment(schedule, currentTime);
  
  // Hitung progress dalam segmen tersebut
  const progress = calculateProgress(currentSegment, currentTime);
  
  // Interpolasi posisi
  const position = interpolate(
    currentSegment.from.coordinates,
    currentSegment.to.coordinates,
    progress
  );
  
  return position;
}
```

**Features:**
1. **Jadwal**: Input jadwal shuttle (JSON/Excel)
2. **Map**: Peta kampus dengan rute shuttle
3. **Marker**: Bus yang "bergerak" sesuai jadwal
4. **ETA**: Estimasi tiba di halte berikutnya

**Tech Stack:**
- **Frontend**: HTML + JavaScript + Leaflet.js
- **Data**: JSON file dengan jadwal
- **Hosting**: GitHub Pages/Vercel (gratis)
- **Update**: JavaScript setInterval untuk update posisi setiap 30 detik

### Advanced: Smart Delay Detection

**Features tambahan:**
- **Manual Delay Input**: Admin bisa input kalau shuttle delay
- **Automatic Adjustment**: Recalculate posisi kalau ada delay
- **Notification**: Notifikasi ke pengguna kalau ada perubahan jadwal
- **Historical Data**: Track on-time performance

## Your Blueprint (Exercise)

### Identifikasi Transportasi Scheduled di Sekitarmu

**Apa yang punya jadwal fixed di kampus/kotamu?**

1. **Campus Shuttle**
   - Bus antar fakultas dengan jadwal tetap
   - Loop route dengan interval reguler
   - Jadwal bisa dilihat di website kampus

2. **Public Transport**
   - Bus kota dengan jadwal (misal: Transjakarta, Trans Semarang)
   - Kereta komuter (KRL, LRT, MRT) - jadwalnya ketat
   - Bus antar kota dengan schedule fixed

3. **Other Scheduled Services**
   - Delivery route (kurir dengan rute harian tetap)
   - Maintenance vehicle (petugas kebersihan dengan jadwal)
   - Security patrol (petugas security patroli berkala)

### Apply Schedule-Based Simulation Pattern

**Untuk transportasi pilihanmu:**

1. **Data Collection**: Kumpulkan jadwal lengkap
   - Waktu berangkat dari setiap stop
   - Waktu tiba di stop berikutnya
   - Koordinat GPS setiap stop

2. **Algorithm**: Buat interpolasi sederhana
   - Hitung total waktu per segmen
   - Hitung posisi berdasarkan waktu yang sudah lewat
   - Update setiap 30-60 detik

3. **Visualisasi**: Tampilkan di peta
   - Marker yang bergerak halus
   - Jalur rute yang jelas
   - Info ETA ke stop berikutnya

4. **Validation**: Test dengan data real
   - Bandingkan posisi simulasi dengan kenyataan
   - Adjust algorithm kalau perlu
   - Tambah fitur delay detection

### Pattern Comparison

**Schedule-Based vs GPS-Based:**

| Aspek | Schedule-Based (168Railway) | GPS-Based (Gojek/Grab) |
|-------|---------------------------|----------------------|
| Hardware | Tidak perlu | Perlu GPS device |
| Accuracy | Tinggi (jika on-time) | Sangat tinggi (real-time) |
| Cost | Rendah (software only) | Tinggi (hardware + data) |
| Works Offline | Ya (data jadwal lokal) | Tidak (butuh GPS signal) |
| Delay Handling | Manual update | Automatic |
| Best For | Scheduled transport | On-demand transport |
